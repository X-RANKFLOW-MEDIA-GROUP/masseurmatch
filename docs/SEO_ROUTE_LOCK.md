# SEO Route Lock

## Purpose

MasseurMatch depends on programmatic SEO. Route indexability must be explicit, tested, and protected from accidental middleware changes.

## Canonical indexable routes

The following route families are SEO assets and must not receive `noindex` unless intentionally retired:

- `/`
- `/{city}`
- `/{city}/{segment}`
- `/{city}/{segment}/{keyword}`
- `/therapists/{slug}` or current public therapist profile equivalent
- Editorial/blog routes that are published
- Comparison or landing pages intentionally built for search

## Non indexable routes

The following route families are UX or account routes and should not be SEO landing pages:

- `/login`
- `/register`
- `/forgot-password`
- `/reset-password`
- `/pro/*`
- `/admin/*`
- `/client/*`
- `/api/*`
- Search/filter URLs with crawlable query parameters

## Explore route rule

`/explore` and `/explore/*` may use `X-Robots-Tag: noindex, follow` when they are browse/search UX pages.

This rule must never be generalized to canonical city or programmatic landing routes.

Examples that must remain eligible for indexation when implemented:

- `/miami`
- `/key-west`
- `/new-york`
- `/miami/outcall-massage`
- `/key-west/hotel-massage`

## Canonical rules

- Legacy `/city/{slug}` and `/cities/{slug}` should 301 to canonical `/{city}` routes.
- Query language variants must redirect to clean canonical URLs.
- Sitemap must only include canonical, intended URLs.
- Robots must not block canonical SEO routes.

## Required tests

- `/explore` returns noindex when expected.
- Canonical city route does not return noindex.
- Sitemap validates.
- Robots validates.
- Legacy city routes redirect permanently.

## Forbidden patterns

- Broad middleware noindex rules that match SEO routes.
- Sitemap URLs that redirect.
- Canonical URLs pointing to non canonical paths.
- Thin autogenerated pages without useful content blocks.
